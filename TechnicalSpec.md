🛠️ Project ARC: Technical Specifications (POC/MVP)1. System ArchitectureThe application follows a Modular Layered Architecture to ensure the UI can be swapped or updated without touching the core logic.Database Layer (database.py): Handles raw SQLite execution, connection pooling, and schema initialization.Service Layer (service.py): Contains business logic (e.g., calculating "Top 10", formatting "NONE" strings, and timestamp generation). This is the primary target for TDD.UI Layer (ui.py): Built with customtkinter. It handles user input, verification modals, and event loops.2. Functional Constraints & LogicSearch Behavior: Searching an employee_id must return a list of previous timestamps and notes. If the list is empty, the UI must explicitly display "NONE".Input Validation: The "Save" action is disabled unless Recorded By is filled and the "Call-Out" checkbox is toggled.Verification Loop: A mandatory modal must appear summarizing the Name, ID, and Notes before the INSERT command is finalized.Admin Policy: No "Edit" or "Delete" functionality exists in the UI. All data corrections are performed via direct SQL manipulation by IT/Admin.4. TDD & Testing StrategyBefore UI development, the AttendanceService must pass an extensive pytest suite.Mocking: Use :memory: SQLite databases for all tests to ensure zero-footprint execution.Coverage Goals:Duplicate EmployeeID rejection.Accurate "Top 10" sorting (Frequency-based).Correct "NONE" string return for new records.System clock/timestamp accuracy for entries.5.
